🗡️ 1. Attack Injection

  • Simulates various prompt injections (e.g. naive, completion, hackaprompt) or adversarial suffixes (gcg).
  • Injected directly into the input block, simulating untrusted user data, RAG context, or external tool outputs.

🧱 2. StruQ Framing

  • Combines the trusted instruction with the untrusted input using reserved special delimiters (e.g., [INST], [INPT], [RESP]).
  • Optional recursive filtering strips out delimiter tokens inserted maliciously by the attacker.

📍 3. Span Mapping

  • Parses the formatted string to definitively categorize each token as PROMPT, DATA, or DELIMITER.
  • Prevents ambiguous boundaries and ensures mitigations only affect untrusted context.

🚦 4. Routing & Mitigation

  • Router: Pools hidden states over DATA tokens only, classifying as BENIGN, PAIR, or GCG using your scikit-learn model.
  • Mitigation: Applies targeted noise or saliency-shrinking strictly to the DATA embeddings, preserving the model's fundamental instruction-following pathways.
